Winsupply - Purchase Orders

One of the applications I worked on at Winsupply was the redesigned Purchase Orders application. This application is designed for users who need to order new product to stock in their warehouses and stores so it is ready to be sold to their customers.

How It Started

This application began as a basic web application ported over from an old green screen application. Most of the application we are modernizing begin life like this. Users often prefer it like this because it means they can quickly navigate using only a keyboard. This is one of several screens used to create purchase orders. The old version is on the left, the new version on the right

The Redesign

Throughout the process of redesigning this application, I created and kept updating a user journey map, created with Coggle. This map charts the user's journey from the moment they log in through all the various parts of the application, including creating worksheets, searching for products, and creating and sending a purchase order to vendors. Feel free to navigate the user journey below.

We are updating our apps for modern devices, built with modern technology like Angular. The old application seen above looks like this now. It's responsive, can be used with a keyboard, mouse, or on a touch screen. And it uses many of the components originally created in the Winsupply Pattern Library.
